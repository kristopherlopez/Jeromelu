# Experience Architecture

## Core Experience
Visitors watch Jeromelu operate.

The product behaves like a public operating theatre for an AI sports character. Users observe:
- thoughts
- actions
- plans
- predictions
- outcomes

The feed is the product.
Everything else supports it.

## Core Surfaces

### 1. Live Feed
A rewindable event timeline showing:
- new source ingestions
- extracted opinions
- detected narrative shifts
- internal reasoning steps
- predictions
- trade planning
- final actions

This is near-real-time, not true streaming.

### 2. War Room
A visualisation layer that makes Jeromelu feel alive between major decisions.

Shows:
- which sources are being processed
- what topics are rising
- consensus movement
- candidate players under review
- current strategic plans

### 3. Team Dashboard
Public SuperCoach team including:
- squad
- trades
- captain choices
- weekly scores
- season rank
- rationale per action

### 4. Opinion Radar
Aggregates ecosystem opinions across:
- YouTube creators
- podcasts
- websites
- blogs
- radio transcripts where available

Tracks:
- buy / sell / hold sentiment
- player mentions
- matchup narratives
- contrarian picks
- consensus shifts over time

### 5. Prediction Ledger
Public record of:
- Jeromelu predictions
- external expert predictions
- eventual outcomes
- accuracy leaderboards

This is one of the most defensible and viral surfaces if handled carefully.

### 6. Ask Jeromelu
Chat interface for personalised SuperCoach advice.

Users can:
- ask team questions
- submit text-based squads
- ask player or matchup questions
- challenge Jeromelu's calls

A temperature control adjusts whether the reply is straight, sharp, or lightly roasting.

### 7. Knowledge Explorer
Browsable pages for:
- players
- teams
- experts
- matchups
- prediction histories
- quotes and source lineage

### 8. Articles (Phase 2)
Weekly content used for:
- SEO / AEO
- authority
- summarising key narratives

# Product Loops

## Daily Loop
intel ingestion -> narrative shift -> Jeromelu reaction -> user validation

Users return to see whether the ecosystem moved and how Jeromelu interprets it.

## Weekly Loop
analysis build-up -> recommendation formation -> final calls -> match review -> postmortem

## Seasonal Loop
prediction record -> rank movement -> expert comparison -> character arc

The strongest retention mechanic is not articles.
It is public continuity.
